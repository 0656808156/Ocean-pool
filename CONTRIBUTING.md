# Contributing to Ocean Pool

We welcome contributions to the Ocean Pool project! To ensure a smooth contribution process, please follow these guidelines:

## Reporting Issues
- Before reporting an issue, please check if it has already been reported.
- To report a new issue, go to the [issues page](https://github.com/0656808156/ocean-pool/issues) and create a new issue with a clear title and description. Include details such as:
  - Steps to reproduce the problem
  - Expected vs. actual behavior
  - Screenshots or error messages, if applicable

## Submitting Pull Requests
1. **Fork the repository**: Click the "Fork" button on the top right corner of the repository page to create your own copy of the repository.
2. **Create a new branch**: Before making any changes, create a new branch for your feature or bug fix. Branch naming should be descriptive, e.g., `feature/add-user-authentication` or `bugfix/fix-navigation-issue`.
   ```bash
   git checkout -b your-branch-name
   ```
3. **Make your changes**: Implement your feature or fix the issue in your local repository.

4. **Test your changes**: Ensure that your changes do not break existing functionality and add tests where applicable.

5. **Commit your changes**: Use clear and concise commit messages. Example:
   ```bash
   git commit -m "Add user authentication feature"
   ```

6. **Push your changes**: Push your changes to your forked repository:
   ```bash
   git push origin your-branch-name
   ```

7. **Create a pull request**: Go to the original repository and click on "New Pull Request". Select your branch and provide a clear description of your changes and the issue it addresses.

## Code Style Guidelines
- Follow the project's coding standards and style guides provided in the documentation.
- Use meaningful variable and function names.
- Keep lines of code to a maximum of 80 characters.
- Write comments where necessary to explain complex logic.
- Ensure proper indentation and formatting according to the project's standards.

## Contribution Workflow
1. **Discuss Before You Code**: If you are unsure about a feature or change, please discuss it in the issues or create a new thread.
2. **Code Review**: Once your pull request is submitted, it will be reviewed by the maintainers. Be prepared to make adjustments based on feedback.
3. **Be Respectful**: Treat others with respect and be open to constructive criticism.
4. **Celebrate Your Contributions**: Once your contribution is merged, celebrate your hard work and share your contribution with the community!

Thank you for contributing to the Ocean Pool project! Your help is greatly appreciated!
